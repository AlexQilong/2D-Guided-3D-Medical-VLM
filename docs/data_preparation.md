# Data Preparation

The README shows the download + prep commands. This page documents the
expected on-disk layout and the CSV schema the training scripts read.

## Directory layout

```
./data/
├── M3D_Cap_npy/
│   ├── ct_case/                       # CT volumes (.npy), shape (D,H,W) or (1,D,H,W), normalized [0,1]
│   ├── ct_quizze/                     # VQA volumes
│   └── M3D_Cap.json                   # GT report annotations (used for GT baselines)
│
└── Duke-Breast-Cancer-MRI/
    ├── preprocessed/                  # MRI volumes (.npy)
    ├── Segmentation_Masks_NRRD/       # GT masks (.nrrd)
    ├── Clinical_and_Other_Features.xlsx
    ├── Annotation_Boxes.csv
    └── {train,val,test}_patient_ids.csv

./checkpoints/                         # see README install table
├── M3D-CLIP/pretrained_ViT.bin
├── M3D-LaMed-Phi-3-4B/mm_projector.bin
├── SegVol/pytorch_model.bin
└── medsam_vit_b.pth
```

`./finetune_data/` and `./outputs/` are generated by the prep scripts (gitignored).

## Notes per dataset

**M3D-Cap** — no preprocessing beyond placing the `.npy` volumes; the report pipeline
consumes them directly. Caption training reads the raw volumes, `M3D_Cap.json` (GT
baselines), and the pseudo-summaries JSON from `scripts/01_generate_pseudo_reports.sh`.

**Duke** — convert DICOM → `.npy` under `preprocessed/<Patient_ID>_volume.npy` (paper
resizes to `1×32×256×256`). GT NRRD filenames must encode the patient id, e.g.
`Segmentation_Breast_MRI_001_Breast.seg.nrrd`. `data_prep/prepare_duke.py` reads the 100
GT patients, splits 50 train / 50 test, resizes volumes, converts NRRD → `.npy`, and
writes the train/test CSVs. `data_prep/prepare_train_csv.py --n_pseudo {100,500}` then
mixes the 50 GT rows with pseudo-mask rows from the remaining patients.

## CSV schema

All segmentation train/test CSVs share this schema:

| Column | Meaning |
|--------|---------|
| `Image` | Relative path to the volume `.npy` |
| `Mask` | Relative path to the mask `.npy` |
| `Mask_ID` | Integer class id selected from the mask (typically 1) |
| `Question_Type` | Always `0` for refseg |
| `Question` | Prompt text, e.g. `"Please segment the lesion in this scan."` |
| `Answer` | Target answer containing the `[SEG]` token, e.g. `"The lesion is segmented as shown in [SEG]."` |
