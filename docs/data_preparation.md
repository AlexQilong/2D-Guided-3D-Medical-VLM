# Data Preparation

## Expected directory layout

```
./data/
├── M3D_Cap_npy/
│   ├── ct_case/                       # 30k+ CT volumes (.npy)
│   ├── ct_quizze/                     # VQA volumes
│   └── M3D_Cap.json                   # GT report annotations
│
└── Duke-Breast-Cancer-MRI/
    ├── preprocessed/                  # MRI volumes (.npy)
    ├── Segmentation_Masks_NRRD/       # GT masks (.nrrd)
    ├── Clinical_and_Other_Features.xlsx
    ├── Annotation_Boxes.csv
    └── {train,val,test}_patient_ids.csv

./checkpoints/
├── M3D-CLIP/pretrained_ViT.bin
├── M3D-LaMed-Phi-3-4B/mm_projector.bin
├── SegVol/pytorch_model.bin
└── medsam_vit_b.pth
```

`./finetune_data/` and `./outputs/` are generated by the prep scripts (gitignored).

## M3D-Cap

1. Download `GoodBaiBai88/M3D-Cap` from HuggingFace and extract under `./data/M3D_Cap_npy/`. The CT volumes must be `.npy` arrays with shape `(D, H, W)` or `(1, D, H, W)`, normalized to `[0, 1]`.

2. The pseudo-report pipeline (`scripts/01_generate_pseudo_reports.sh`) takes the raw `.npy` volumes as input and produces a JSON of per-volume summaries. No further preprocessing is needed for the caption track.

3. For caption training, `lamed/dataset/multi_dataset.py` reads:
   - the raw volume `.npy` files
   - the GT annotations from `M3D_Cap.json` (used at small scale for the GT-trained baselines)
   - the pseudo summaries JSON produced in step 2

## Duke Breast Cancer MRI

1. Download from TCIA: <https://doi.org/10.7937/TCIA.e3sv-re93>. There are two parts:
   - The main DICOM dataset (922 patients)
   - The segmentation supplement (`PKG-Duke-Breast-Cancer-MRI-Supplement-v3`) with 100 expert-annotated NRRD masks

2. Convert DICOM → `.npy` and place under `./data/Duke-Breast-Cancer-MRI/preprocessed/<Patient_ID>_volume.npy`. The paper resizes each volume to `(1, 32, 256, 256)`.

3. Place GT NRRD masks under `./data/Duke-Breast-Cancer-MRI/Segmentation_Masks_NRRD/`. Filenames must encode the patient id, e.g. `Segmentation_Breast_MRI_001_Breast.seg.nrrd`.

4. Prepare the train/test CSVs:

```bash
python data_prep/prepare_duke.py
```

This:
- Reads the 100 patients that have GT masks
- Splits them 50 train / 50 test (matches the paper's protocol)
- Resizes volumes and converts NRRD → `.npy` masks
- Writes:
  - `./finetune_data/duke_volumes/<Patient_ID>_volume.npy`
  - `./finetune_data/converted_masks/<Patient_ID>_gt_mask.npy`
  - `./finetune_data/duke_gt/train.csv` (50 rows, one per GT patient)
  - `./finetune_data/duke_test.csv` (50 rows, held out)

5. Generate pseudo masks for the remaining 822 patients (no GT):

```bash
bash scripts/02_generate_pseudo_masks.sh \
    ./data/Duke-Breast-Cancer-MRI/preprocessed \
    ./outputs/duke_pseudo_masks_medsam \
    "breast tissue" \
    ./checkpoints/medsam_vit_b.pth \
    vit_b
```

6. Build the GT + pseudo training CSVs (used by the GT+100Pseudo and GT+500Pseudo conditions):

```bash
python data_prep/prepare_train_csv.py --n_pseudo 100   # → finetune_data/duke_gt_plus_pseudo/train.csv
python data_prep/prepare_train_csv.py --n_pseudo 500   # → finetune_data/duke_gt_plus_pseudo_500/train.csv
```

## CSV schema

All training/test CSVs share this schema:

| Column | Meaning |
|--------|---------|
| `Image` | Relative path to the volume `.npy` |
| `Mask` | Relative path to the mask `.npy` |
| `Mask_ID` | Integer class id selected from the multi-class mask (typically 1) |
| `Question_Type` | Always `0` for refseg |
| `Question` | Prompt text (e.g. `"Please segment the lesion in this scan."`) |
| `Answer` | Target answer including the `[SEG]` token (e.g. `"The lesion is segmented as shown in [SEG]."`) |
